Data Backup & Recovery

All data lives in PostgreSQL on this server PC. POS terminals are just clients — if a terminal breaks, install on a new PC and connect.

If the server PC is damaged: restore PostgreSQL on a new PC, then restore from backup below. All data recovered instantly.

Recovery instructions:
1. Install PostgreSQL on new server PC  →  2. Create DB + user (see Admin Guide)
3. Start API server (npm start in server/)  →  4. Go to Settings → Restore & upload backup file
5. Reinstall POS on terminals → connect to new server IP → all data immediately available

Auto Update

POS terminals check for updates automatically on startup and every 4 hours. When a new .exe is built:

  1. Run npm run build in pos/
  2. Copy dist/POS Setup X.X.X.exe + latest.yml to server/updates/
  3. Terminals detect update automatically and prompt user to restart
